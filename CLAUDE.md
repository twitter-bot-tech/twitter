# MoonX Agent Twitter — Claude 使用规范

## 元提示（Meta Prompt）
每次新会话开始时，自动加载以下上下文文件（如存在）：
- `claude-context/about-me.md` — 用户身份与沟通偏好
- `claude-context/brand-voice.md` — MoonX 品牌语气规范

**行为准则**：
- 始终以 Kelly（MoonX 营销负责人）的视角和需求为中心
- 不熟悉的需求先查阅上述上下文文件，再回答
- 任务结束后，若有新的项目进展，提示用户是否更新 `current-projects.md`

## 环境与计费
- 本项目使用 Claude Code **Pro 订阅**，不使用 API Key 计费
- 禁止在 .env 文件中添加或修改 `ANTHROPIC_API_KEY`
- 遇到计费问题，确认 Pro 订阅是否激活，不要引导用户检查余额或 API 额度

## 沟通规范
- **默认使用中文回复**，除非用户切换为英文
- 用户提到 UI 元素、页面、文档名称时，先追问具体文件路径或截图，不要猜测
- 回复简洁直接，不用 emoji

## 提问格式规范（需要 Kelly 做决策时必须遵守）
每次向 Kelly 提问，必须按以下四步结构，假设她已经 20 分钟没看这个窗口：
1. **重新定位**：1-2 句说明当前任务/背景
2. **简化**：用具体例子解释问题，不用术语和内部概念，说它「做什么」而不是「叫什么」
3. **推荐**：`推荐选 [X]，因为 [一句话理由]`
4. **选项**：用字母标注 A / B / C

每个问题单独提问，等 Kelly 回复后再继续，不要一次列出多个问题。

## 工作流规则（重要）
- **先确认再执行**：提案营销方案、活动计划或任何大段文档前，先列出 ≤5 条要点大纲，等用户明确确认方向后，再开始写完整文档或代码
- 任何配置变更完成后，用 grep 全局搜索确认没有遗漏引用，确认完才算完成
- 多步骤任务每完成一步主动汇报进度，不要等全部完成再一次性输出

## 两将军体系（Agent 路由规则）
Kelly 的营销团队按以下层级运作：

```
Kelly（最终决策）
    │
    ├── 增长将军：社媒运营 + KOL媒体   → 拉新 / 曝光 / 内容
    │
    └── 运营将军：SEO + 增长 + 策略数据 → 留存 / 转化 / 数据
```

**任务路由原则：**
- 新方案启动前，先走 `/marketing-plan` 的第零步过滤，确认方向对再执行
- 增长类任务（KOL 外联、社媒内容、传播活动）→ 增长将军范畴
- 数据/优化类任务（关键词、用户分析、漏斗优化）→ 运营将军范畴
- 跨将军任务，由 Kelly 直接协调，不绕路

## 推文发布规范（基于数据）

- **禁止连续 Thread 格式**：Thread 子推曝光仅为独立推文的 1/5，一律改为独立推文分开发
- **禁止纯 CTA 推文**：「Follow @moonx_bydfi」类纯 CTA 推文 0 互动，已证明无效，取消
- **每条推文必须有钩子**：数据陈述后加一句开放问题或反常识追问，不能只摆数字就结束
