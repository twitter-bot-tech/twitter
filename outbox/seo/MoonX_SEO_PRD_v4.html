二、需求背景

2.1 需求价值

MoonX 当前自然流量为 0，bydfi.com 整站 28 天自然流量 46.1K，MoonX 无任何 SEO 内容承接。核心问题：

  • MoonX 全部页面为 CSR 渲染，Google 无法抓取内容，零自然流量
  • 没有内容落地页，用户无法通过搜索发现 MoonX
  • 竞品 laikalabs.ai（DA 仅 19）已开始布局，存在先发危机

本需求价值：① 建立 MoonX 内容 SEO 基础设施，目标 6 个月月均自然流量 15,000+；② 以内容集群方式建立"预测市场聚合器"品牌权威；③ 程序化页面形成规模化长尾流量。

2.2 目标用户

站外自然搜索流量用户，典型场景：

  1. 搜索 polymarket vs kalshi（1,700/月）进入对比文章 → 了解 MoonX 聚合两者 → 跳转 MoonX 平台
  2. 搜索 TSLA prediction market odds 进入美股预测页 → 看到 Polymarket 上 TSLA 相关市场 → 跳转交易
  3. 搜索 PEPE price prediction 进入 Meme 购买页 → 看到预测市场赔率 → 跳转 MoonX 下注
  4. 搜索 top prediction markets today 进入热门榜单 → 发现感兴趣市场 → 点击参与

五、需求详情

通用技术要求（所有 SEO 页面）：
① 所有 SEO 页面必须 SSR 渲染，Google 爬虫能直接读取 HTML 内容
② 每个页面必须有独立的 <title>、<meta description>、<canonical>
③ 所有内链使用 <a href> 标签，不使用 JS 跳转
④ 页面 LCP ≤ 2.5s（Core Web Vitals 达标）

5.2 程序化 SEO 页面

程序化页面通用规则：
① 所有模板变量用 {变量名} 标注，服务端渲染时自动填充
② Title / H1 / Meta 均为程序化生成（模板 + 变量），每页唯一
③ 所有价格/赔率数据：SSR 生成静态快照（Google 可读），CSR 负责实时更新
④ noindex 规则：事件已结束 >30天 / Token 已 rug / 数据缺失 → 自动加 noindex

6.1 人工内容页面验收

  • 访问 /en/moonx/learn/prediction/ 列表页，查看页面源代码，能看到完整 HTML 文章列表（SSR 验证）
  • 访问 /en/moonx/learn/prediction/polymarket-vs-kalshi，源代码包含完整文章内容（SSR 验证）
  • <title> 为「Polymarket vs Kalshi: Full Comparison 2026 | MoonX」
  • <meta name="description"> 存在且 ≤ 155 字符
  • <link rel="canonical"> 正确指向当前 URL
  • 页面包含 Article Schema（用 Google Rich Results Test 验证，无错误）
  • 页面包含 FAQPage Schema（有 FAQ 模块时）
  • 页面包含 BreadcrumbList Schema
  • FAQ 折叠 / 展开功能正常，内容可见
  • 文章内链正常跳转，无 404
  • GSC URL Inspection → 「页面已编入索引」
  • CMS 发布文章后，前端实时生效，无需重新部署
  • SEO Title 超60字符时，CMS 后台实时红色警告

6.2 程序化页面验收

  • 访问 /en/moonx/markets/stocks/TSLA，源代码包含 TSLA 相关 Polymarket 市场数据（SSR 验证）

  • sitemap_moonx_tokens.xml 包含满足准入条件的 Token URL，不包含低质量 Token
  • 热门榜单页市场卡片数据在30分钟内完成一次 CSR 更新

  • 程序化页面 Title 不同市场/Token 均唯一，无重复
