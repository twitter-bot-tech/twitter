1.2 前端实现通则

约定项规则
SSR 框架Next.js ISR（Incremental Static Regeneration）。每个页面通过 revalidate 控制重建周期，revalidate 值见各页面章节。SSR 构建失败时自动 stale cache 兜底，用户不感知。
CSR 轮询使用 setInterval 或 SWR。请求失败时 silent fail：保留当前显示值不变，不弹出错误提示。下一轮次再重试，连续失败3次后停止轮询并在 console 告警。
SSR 数据注入通过 getStaticProps / getServerSideProps 返回 props，注入页面组件。不使用全局 window 变量传 SSR 数据。
<a href> 规范所有 SEO 内链必须使用原生 <a href="..."> 标签（Next.js <Link> 组件会正确渲染为 <a>）。禁止用 onClick + router.push 替代，Google 爬虫不执行 JS 点击事件。
noindex 注入时机在 getStaticProps 中计算 end_date 与当前时间差；距今 > 30 天则在 props 中返回 noindex: true，页面组件据此在 <Head> 中插入 <meta name="robots" content="noindex">。
Skeleton 加载态CSR 数据首次加载期间展示骨架屏（灰色占位条，animate-pulse）。SSR 已有初始数据的模块不需要 skeleton，只有纯 CSR 模块才需要。
接口字段缺失处理所有字段在渲染前做空值判断。字段缺失时降级显示「—」或隐藏该 UI 元素，不抛 JS 异常。

3. 调用 revalidatePath purge 缓存
noindex 触发（内容过期） 1. 文章响应中 noindex=true，前端注入 <meta name="robots" content="noindex">
2. 从 Sitemap 移除（已 noindex 的 URL 不应出现在 Sitemap）
3. 不删除页面（保留入站链接），仅停止 Google 索引

16.4 降级策略（API 失败处理）

与 PRD 5.3 降级规则一致，以下为前端实现要点。
数据类型一级降级二级降级缓存有效期
文章正文 / 字段展示 stale cache 内容 + 顶部提示条显示"内容加载中"骨架屏；不加 noindexISR stale-while-revalidate
Meme 侧栏 Token 价格展示上次缓存价格 + 时间戳隐藏价格行，侧栏其他信息正常展示5 分钟
Meme 侧栏赔率展示缓存赔率 + "数据更新于 X 分钟前"隐藏「See Odds」CTA，改为纯 Token 详情链接60 分钟

关键原则：Title / H1 / Meta 均来自 CMS 静态字段，不依赖实时 API；任何 API 连续失败 >15 分钟触发 Lark 告警。
