一、需求背景

1.1 现状与核心痛点

MoonX 是 BYDFi 旗下的预测市场聚合器产品，当前面临自然流量为零的关键增长瓶颈：

  • MoonX 全部页面为 CSR 渲染，Google 爬虫无法读取页面内容，导致零自然搜索流量
  • 没有任何内容落地页，用户无法通过自然搜索发现 MoonX
  • 母站 bydfi.com 整站 28 天自然流量仅 46.1K，SEO 基础薄弱
  • 竞品 laikalabs.ai（DA 仅 19，极弱）已开始内容布局，存在先发危机
核心风险：若不在 3 个月内建立 SEO 内容基础设施，MoonX 将在"预测市场聚合器"这一新兴品类关键词竞争中失去先机，流量红利将被先行者收割。

通用技术要求（所有 SEO 页面）：
① 所有 SEO 页面必须 SSR 渲染，Google 爬虫能直接读取 HTML 内容
② 每个页面必须有独立的 <title>、<meta description>、<canonical>
③ 所有内链使用 <a href> 标签，不使用 JS 跳转
④ Core Web Vitals 三项全部达标：LCP ≤ 2.5s（最大内容绘制）/ INP ≤ 200ms（交互响应，2024 年起纳入排名因子）/ CLS ≤ 0.1（布局偏移，图片必须写 width/height 防止 reflow）
⑤ TTFB ≤ 800ms（服务端响应时间，SSR 页面关键指标，超出 Google 会降低爬取频率）
⑥ 图片规范：所有图片必须添加 width/height 属性 + alt 文本；首屏以下图片加 loading="lazy"；优先使用 WebP 格式
