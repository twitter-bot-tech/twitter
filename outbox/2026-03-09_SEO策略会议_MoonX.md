## 二、五大致命问题

### 问题 1：整站是 SPA，Google 爬不到任何内容 ⚠️ 最高优先级

**实测结果：**
抓取 `https://www.bydfi.com/en/moonx/markets/trending` 只能获得 CSS 变量和主题配置，**没有 H1、没有正文、meta description 为空字符串**。

**影响：**
Google 看到的 MoonX 页面是一张白纸。即使有用户流量，也完全无法被索引排名。

**根因：**
React/Vue SPA 纯客户端渲染，爬虫不执行 JavaScript（或执行不完全）。

**解法：**
- 短期：给核心落地页加 SSR（服务端渲染）或 SSG（静态生成）
- 最低成本替代方案：在 `<head>` 里注入静态 meta + 在页面底部注入静态 HTML 文本块（对爬虫可见）
- 工期评估：需要找技术团队，**这是 SEO 的前提条件，不做其他都白搭**

## 五、发布平台策略

### 选择 A：bydfi.com/blog/（推荐，长期最优）
- **优点：** 权重全部留在主域名，内链直接指向产品页，复利效应最强
- **缺点：** 需要技术团队开发博客模块（1~2周），需要 SSR 配合
- **结论：** 这是正确路径，必须建

### 选择 B：Medium（已选，短期启动）
- **优点：** 0 开发成本，本周内可发，DA 高（Medium 本身权重）
- **缺点：** 权重不回流 bydfi.com，长期没有积累价值
- **结论：** 作为 3/20 前快速占位用，Pillar Page 已写好，直接发

### 选择 C：Ghost/Substack 独立博客（不推荐）
- 权重分散，维护成本高，不如直接建 bydfi.com/blog

**执行建议：**
1. 本周：Medium 发 Pillar Page（polymarket alternative），先占位
2. 3/20 前：推动技术团队在 bydfi.com 下开 `/blog/` 路由，支持 SSR
3. 3/20 后：所有新内容发 bydfi.com/blog，Medium 文章加 canonical 指向主域
