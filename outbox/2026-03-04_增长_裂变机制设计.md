**项目：** MoonX — 预测市场聚合 + 聪明钱追踪
**设计者：** 增长运营（Growth Lead）
**日期：** 2026-03-04
**目标：** Q1 注册用户 1000（加密 700 + 美股 300）

---

## 一、机制核心逻辑

### 设计哲学

三条不妥协的原则：

1. **每个激励必须有防刷设计**（Blur 积分系统的核心教训）
2. **飞轮不靠补贴，靠稀缺感和社会排名**（Uniswap 空投的核心驱动力）
3. **用户的「下一步行动」永远清晰**（减少认知负担，提高转化）

### 核心机制名称：MoonX Priority Pass

用户注册即加入等待名单，拿到一个排队号。
排队号本身没有价值，**邀请行为才能改变命运**。
这不是简单的「邀请有奖」，而是「**越早行动，永久优势越大**」。

---

## 二、激励层次设计（Tier System）

参考 Blur 积分分层逻辑，设计三层结构：

| Tier | 条件 | 权益 | 积分加成 |
|------|------|------|----------|
| Waitlisted（等待中） | 0 有效邀请 | 基础积分 100 pts，排队等待 | 1.0x |
| Early Bird（早鸟） | 邀请 3 人完成首次登录 | 提前进入 + 「Early Bird」永久标签 + 200 pts 奖励 | 1.0x |
| Power User（权力用户） | 邀请 10 人完成首次登录 | 永久积分加成 +20%（1.2x 乘数）+ 专属 KOL 数据看板入口 + 500 pts 奖励 | 1.2x |

**关键设计细节：**
- 积分乘数是**永久生效**，不是活动期限制，这是真实长期价值
- 「KOL 数据看板」是 MoonX 聪明钱追踪的核心差异化功能，这对目标用户（链上玩家）极具吸引力
- Early Bird 标签在用户档案永久显示，制造身份认同

---

## 三、防刷机制设计

### 核心规则：有效邀请 = 被邀请人完成「首次登录」

- 注册不算：必须登录一次才计入邀请人的有效邀请数
- 同设备注册：建议后续加设备指纹检测（目前阶段用登录确认代替）
- 邀请自己：code 与 email hash 绑定，无法自我邀请

### 分层防刷逻辑

```
注册时：invitee → pending_invites +1（等待确认）
首次登录后：pending → valid_invites +1，积分才正式发放
```

**后续增强方向（V2）：**
- 要求被邀请人完成 1 次交易/预测操作才算有效（激活深度校验）
- 邮箱域名黑名单（一次性邮箱过滤）
- 同 IP 注册上限（每 IP 每天最多 3 次）

## 四、用户旅程图

```
[发现 MoonX]
      │
      ▼
[访问等待名单注册页]
      │
      ▼
[输入邮件 + 姓名 → 可选：填写邀请码]
      │
      ▼
[获得专属邀请码 + 排队号 #XXX]
[+100 基础积分]
      │
      │── 不分享 ──→ [等待开放 · 队列无变化]
      │
      ▼
[分享邀请链接到 Twitter / Telegram / Discord]
      │
      ▼
[好友通过链接注册]
      │       ← pending 状态，暂不计入有效邀请
      ▼
[好友完成首次登录] ← 关键节点（防刷门槛）
      │
      ▼
[邀请人：valid_invites +1，+50 pts]
      │
      ├── valid_invites == 3 → Early Bird 解锁！+200 pts + 永久标签
      │
      └── valid_invites == 10 → Power User 解锁！+500 pts + 1.2x 乘数 + KOL 看板
```

**用户的「Aha Moment」：**
- 第一次看到自己的排队号 + 邀请链接 → 「我有了专属通行证」
- 第一次有好友登录确认 → 「积分涨了，标签升级了」→ 社交正反馈

## 五、自我加速（飞轮）机制

```
更多用户加入
    → 排队号数字变大
    → 新用户感受到「稀缺感」和「紧迫感」
    → 新用户更有动力分享（提前进入的动机更强）
    → 更多邀请发出
    → 更多用户加入（循环加速）
```

**社会比较效应：** Leaderboard 公开显示邀请排名，激发竞争心理（类似 Blur 的 Season 积分排行榜）。

**身份驱动：** Early Bird / Power User 标签是产品内稀缺的社会货币，不可购买，只能通过邀请赢得。

## 六、技术实现方案

### 架构：零改动现有产品，100% Python 脚本 + CSV/Excel 追踪

**文件结构：**
```
04_growth/
├── generate_referral_codes.py      # 主程序
└── referral_data/
    ├── waitlist_users.csv           # 用户表（核心数据）
    ├── referral_links.csv           # 邀请关系追踪
    ├── activity_log.csv             # 操作日志
    └── MoonX_Waitlist_Dashboard.xlsx  # 导出的 Excel 看板
```

### 命令行操作手册

```bash
# 1. 新用户注册（无邀请码）
python generate_referral_codes.py --action register \
  --email alice@example.com --name "Alice"

# 2. 新用户注册（有邀请码）
python generate_referral_codes.py --action register \
  --email bob@example.com --name "Bob" --ref MOON-ALICE-3F9A

# 3. 确认用户首次登录（触发积分发放）
python generate_referral_codes.py --action confirm_login \
  --email bob@example.com

# 4. 查询用户状态
python generate_referral_codes.py --action status \
  --email alice@example.com

# 5. 查看排行榜
python generate_referral_codes.py --action leaderboard

# 6. 查看整体数据统计
python generate_referral_codes.py --action stats

# 7. 导出 Excel 看板
python generate_referral_codes.py --action export
```

### 数据表结构

#### waitlist_users.csv

| 字段 | 类型 | 说明 |
|------|------|------|
| email | string | 主键 |
| name | string | 显示名 |
| referral_code | string | 专属邀请码（格式：MOON-XXXXX-XXXX）|
| referred_by_code | string | 注册时使用的邀请码（可为空）|
| queue_position | int | 排队号（按注册顺序）|
| registered_at | ISO8601 | 注册时间 |
| first_login_at | ISO8601 | 首次登录时间（空=未登录）|
| valid_invites | int | 有效邀请数（好友已登录）|
| pending_invites | int | 待确认邀请数（好友已注册未登录）|
| points | int | 当前积分 |
| tier | enum | waitlisted / early_bird / power_user |
| tier_upgraded_at | ISO8601 | 最近一次升级时间 |
| status | enum | 同 tier（兼容字段）|

#### referral_links.csv

| 字段 | 类型 | 说明 |
|------|------|------|
| referral_code | string | 使用的邀请码 |
| inviter_email | string | 邀请人 |
| invitee_email | string | 被邀请人 |
| invited_at | ISO8601 | 注册时间 |
| login_confirmed_at | ISO8601 | 登录确认时间（空=待确认）|
| is_valid | 0/1 | 0=pending，1=已确认 |

#### activity_log.csv

| 字段 | 说明 |
|------|------|
| timestamp | 操作时间 |
| email | 操作相关用户 |
| action | register / first_login / referral_pending / invite_confirmed |
| detail | 附加信息（code、积分变化等）|
